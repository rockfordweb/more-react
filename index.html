Setting up your Router

Route

<Route handler={Component}>

Props: name, path, handler, ignoreScrollBehavior

Child routes

Nest your routes like you would nest tags


<Route handler="{App}">
  <Route path="thing-one" handler={ThingOne} />
  <Route path="thing-two" handler={ThingTwo} />
</Route>

DefaultRoute

<DefaultRoute handler={Default} />

Ensures that a RouteHandler is always rendered

NotFoundRoute

<NotFoundRoute handler={RouteNotFound} />

Allows for granular, context-sensitive "not found" pages

Redirect

<Redirect from="old" to="new" />

Instantiate your App

Router.run()

Accepts three arguments:

Routes, as previously defined

Location, optional, defaults to Router.HashLocation

Callback provides Root component and state object

Location

HashLocation (default)

HistoryLocation uses the HTML5 browser history API

RefreshLocation initiates full page reload on transition

Components

Root

This component is created by the callback when you start your router

Render this to start your app

RouteHandler

Call <RouteHandler /> where you would normally put the component for that route

Route Handler

Not to be confused with RouteHandler

URL segments, query parameters and full path are all injected as properties

Link

<Link to="routeName" />

Props: to, params, query, activeClassName, activeStyle, onClick

Flux Architecture

http://facebook.github.io/flux/

Unidirectional Data Flow

Action

Simple object containing some data and a type

Dispatcher

The dispatcher acts as the main interchange of data in your application

Stores

Register callbacks with the dispatcher

Emit change events when updated from the dispatcher

Views

Renders on change events

Handle user interaction, which may result in an action
